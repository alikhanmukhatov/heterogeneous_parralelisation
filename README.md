# Assignment 1 — C++ и OpenMP

## Задания

1. **Динамический массив (50 000 чисел)**  
   - Заполнение случайными числами (1–100)  
   - Вычисление среднего  
   - Освобождение памяти (`delete[]`)  

2. **Последовательный поиск минимума и максимума (1 000 000 чисел)**  
   - Замер времени выполнения  

3. **Параллельный поиск минимума и максимума (OpenMP)**  
   - `#pragma omp parallel for reduction(min:...)` и `reduction(max:...)`  
   - Сравнение с последовательной реализацией  

4. **Вычисление среднего (5 000 000 чисел)**  
   - Последовательно и параллельно (`reduction(+:sum)`)  
   - Сравнение времени  

---

## Контрольные вопросы

1. **Динамический vs статический массив:**  
   - Динамический в куче, размер во время выполнения  
   - Статический в стеке, размер фиксирован  

2. **Указатель:** хранит адрес памяти; нужен для работы с динамическими массивами  

3. **Освобождение памяти:** предотвращает утечки  

4. **Последовательная vs параллельная обработка:**  
   - Последовательная — по одному элементу  
   - Параллельная — несколько потоков одновременно  

5. **`#pragma omp parallel for`:** распараллеливает цикл между потоками  

6. **`reduction`:** безопасное объединение результатов потоков (сумма, минимум, максимум)  

7. **Зачем `reduction` для суммы:** обычная переменная приведёт к гонкам потоков  

8. **Почему параллель может быть медленнее:**  
   - Малые данные  
   - Накладные расходы на потоки  
   - Недостаток ядер  
   - Частые синхронизации
